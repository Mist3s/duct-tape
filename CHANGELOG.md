# История изменений

Формат основан на [Keep a Changelog](https://keepachangelog.com/ru/1.1.0/),
проект придерживается [семантического версионирования](https://semver.org/lang/ru/).

## [1.1.3] — 2026-07-18

### Изменено
- Собираемый EXE и артефакт релиза переименованы в `duct-tape.exe` — в тон
  репозиторию и иконке. Внутренний пакет и CLI-команды остаются `omsreg`.

## [1.1.2] — 2026-07-18

### Исправлено
- В собранном EXE не отображалось ни одной утилиты. Плагины искались через
  файловый скан (`pkgutil.iter_modules`), которого нет в onefile-сборке
  PyInstaller, — список оказывался пустым (из исходников всё работало). Теперь
  встроенные плагины импортируются явно (`BUILTIN` в `plugins/__init__.py`) и
  попадают в сборку; авто-скан папки оставлен для запуска из исходников. В сборку
  добавлен `--collect-submodules omsreg.gui.plugins`.

### Документация
- README: как добавлять плагин (в т.ч. вписывать его в `BUILTIN` для exe); логотип
  над заголовком.

## [1.1.1] — 2026-07-18

### Добавлено
- Иконка приложения «синяя изолента» (`icon.png` / `icon.ico`): показывается в окне
  программы, включена в пакет и в сборку EXE (`--icon`).
- Три скриншота разделов в README.

### Изменено
- Кнопка «Сохранить настройки» прижата к правому краю ряда действий.

## [1.1.0] — 2026-07-18

### Изменено
- Навигация: вертикальный список утилит слева вместо горизонтальной полосы вкладок
  (она переполнялась уже на 4–5 длинных названиях, часть разделов уходила за край).
  Масштабируется на любое число утилит, с прокруткой при необходимости.
- Полосы прокрутки списка и журнала авто-скрываются, когда прокручивать нечего,
  и оформлены в тон интерфейсу.
- Окно расширено (1120×780, минимум 1000×650) — кнопки не обрезаются.
- Высота рабочей области зафиксирована по самой высокой утилите — журнал больше не
  «прыгает» при переключении разделов.
- Описания утилит приведены к единой ширине (авто-перенос вместо ручных переносов).
- Кнопки действий прижаты к низу рабочей области; «Сохранить настройки» перенесена
  в один ряд с ними.

## [1.0.0] — 2026-07-18

Первый публичный выпуск. Глубокий рефакторинг перед публикацией на GitHub:
три отдельных скрипта превращены в пакет-платформу `omsreg`.

### Добавлено
- Пакет `src/omsreg` со слоями `core` / `utils` / `gui`.
- Реестр утилит-плагинов: новая утилита добавляется одним файлом с `SPEC`,
  вкладка интерфейса и хранение настроек строятся автоматически из схемы.
- Упаковка (`pyproject.toml`), точки входа (`omsreg`, `omsreg-gui`, `omsreg-*`),
  `LICENSE` (MIT), тесты, CI (GitHub Actions) и сборка EXE по тегу.
- Единая команда `omsreg` с подкомандами.

### Изменено (безопасность данных)
- Запись DBF стала **атомарной** (временный файл → `fsync` → `os.replace`):
  исходный файл больше не может остаться повреждённым при сбое записи.
- В резервную копию попадает **всё семейство файлов** (`.dbf` + `.fpt/.dbt` +
  `.cdx/.idx`), а после упаковки устаревшие индексы удаляются и сбрасывается флаг
  индекса в заголовке — потребитель перестроит индекс заново.
- Заметное предупреждение в журнале при удалении большой доли записей файла
  (защита от неверного поля/файла кодов).
- CSV статистики использует десятичную запятую (корректный импорт в русский Excel).
- Статистика теперь пишет прогресс в общий журнал (как утилиты удаления).

### Исправлено
- Сборка EXE собирала несуществующий `reestr_gui.py` — устранено переходом на
  точку входа пакета.
- Правило `*.txt` в `.gitignore`, молча прятавшее `requirements.txt`, протоколы и
  примеры, заменено точечными правилами.
- Устранено троекратное дублирование `DbfTable`, `LDID_CODEPAGES`, `JobError`,
  разборщиков значений и настройки логирования — теперь всё в `omsreg.core`.

### Удалено
- Личный `настройки.txt` и IDE-файлы из репозитория; вместо конфига — пример в `examples/`.
